# Technical Research: Text-Heavy LinkedIn Carousels

**Topic**: Generating specific, legible, text-heavy content for LinkedIn Carousels (Technical Implementation).
**Date**: 2026-01-12
**Status**: Completed

## 1. Executive Summary
To support "Text-Heavy" or "Text-Only" carousels while maintaining the "Zero Hallucination" safety standard, we must move away from *Generative AI Text* (asking DALL-E/Imagen to write words) and towards **Programmatic Composition**.

**Recommendation**: Implement a **Hybrid Rendering Pipeline** using `Satori` (HTML-to-SVG) within Supabase Edge Functions.
*   **Backgrounds**: Generated by Imagen 3 (Visuals only, no text).
*   **Text/Layout**: Rendered via HTML/CSS templates overlaid on the background.
*   **Output**: 
    1.  **Images (PNG)**: For standard approval flow.
    2.  **PDF (Future)**: The *Gold Standard* for LinkedIn organic reach in 2025.

## 2. Key Findings

### 2.1 The "PDF First" Reality (2025)
Research confirms that LinkedIn favors **PDF Uploads** ("Documents") for organic carousels.
*   **Algorithm**: High dwell time signals; PDF slides are swipeable and text is selectable.
*   **Format**: Uploading 10 PNGs is treated as a "gallery"; Uploading 1 PDF is treated as a "Carousel".
*   **Implication**: Our system should ideally output a PDF file for the user to download/post.

### 2.2 Text Rendering Technologies
| Tech | Pros | Cons | Verdict |
| :--- | :--- | :--- | :--- |
| **Imagen 3 / DALL-E 3** | Easy integration. Creative typography. | High hallucination rate. Spelling errors. Unpredictable layout. | **REJECTED** for body text. |
| **Ideogram API** | Best-in-class AI text rendering. | Additional cost. Rasterized text (pixels, not vector). Limited layout control. | **Backup Option**. |
| **Satori (Vercel/Deno)** | **Perfect control**. CSS-based styling. Fast. Runs on Edge. | Requires building "Templates" (React/JSX). | **RECOMMENDED**. |
| **Canvas API** | Standard. | Low-level drawing (hard to wrap text, manage fonts). | **Too complex**. |

### 2.3 Proposed Architecture (Satori + Deno)
Supabase Edge Functions (Deno) support `satori` via `https://esm.sh/satori`.

**Workflow:**
1.  **AI (Text)**: Generate Copy (Headline, Body) -> JSON.
2.  **AI (Image)**: Generate Background (Abstract/Clean) -> PNG URL.
3.  **Compositor (Edge Function)**:
    *   Load Font (Inter/Roboto).
    *   Define Layout in simple HTML/JSX (Background Image + Absolute Divs for Text).
    *   `satori(markup) -> SVG`.
    *   `resvg.render(svg) -> PNG`.
4.  **Delivery**: Save componsited PNG to Storage.

## 3. Implementation Plan (Next Steps)
1.  **Prototype**: Create a `generate-slide-image` Edge Function that takes `{ text, background_url }` and returns a composite PNG.
2.  **Templates**: Create 2-3 standard layouts (Text Only, Split Left/Right, Hook Slide).
3.  **PDF Stitching**: Use `pdf-lib` to combine the final PNGs into a downloadable PDF document.
